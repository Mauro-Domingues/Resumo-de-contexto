Configuração geral do git bash

Git config –global user.name "SeuNome" (Configura o nome do usuário/autor dos commits)

Git config –global user.email "SeuEmail" (Configura o email do usuário/autor dos commits)

Git config --global init.defaultBranch main (Configura a branch principal como main)

Fases de vida do arquivo

Untracked o git ainda não conhece a existência do arquivo em nenhuma versão

Unmodified ficam os arquivos que não sofreram nenhuma alteração em relação à última versão

Modified ficam os arquivos que sofreram alteração em relação à última versão

Staged ficam os arquivos que estão prontos para serem inseridos na nova versão

Comandos do git bash no projeto

Git status = Para consultar o status dos arquivos

Git log = Para consultar commits

Git commit -m = O commit é utilizado para avisar o git que queremos gerar um novo estado com os arquivos em staged

Git branch = Para criar uma nova branch

Git checkout = Para alterar de branch

Git merge = Para unir duas branches mais recentes

Git rebase = Para pegar todas as alterações que foram confirmadas em um branch e reproduzi-las em outra

--continue = Às vezes, durante um processo de rebase do Git, você pode encontrar alguns conflitos de fusão e o Git pede que você resolva os conflitos para avançar com a rebaseação/mesclagem

Git log --graph = Para consultar commits visualizando as ramificações

Git checkout = Para alterar de branch/Para voltar o arquivo para antes da edição

Git diff = Para ver as modificações em um arquivo

Git reset HEAD = Para voltar arquivos staged para modified

Git reset --soft = Volta ao commit anterior mantendo as alterações em staged

Git reset --mixed = Volta ao commit anterior mantendo as alterações em modified

Git reset --hard = Volta ao commit anterior removendo todas as alterações do commit

Git remote add origin = Para adicionar um repositório remoto

Git clone = Para clonar um repositório remoto

Git push origin main = Para enviar o código para o repositório remoto

Git pull origin main = Para obter as modificações de um repositório remoto

Git stash --include = Para armazenar uma alteração temporária

Git stash apply = Para aplicar uma alteração temporária

Git stash list = Para listar alterações temporárias

Git stash clear = Para limpar uma alteração temporária

.gitignore = Para ignorar um arquivo que você não queira enviar para o repositório remoto

Git revert = Para desfazer um commit mantendo o histórico

Workflow é um termo inglês que significa fluxo de trabalho

Centralized Workflow = Todos os desenvolvedores trabalham na branch main

Feature Branch Workflow = Todos os desenvolvedores trabalham com novos recursos utilizando uma nova branch

Gitflow Workflow = Todos os desenvolvedores trabalham com um modelo estrito de branches projetado em torno do lançamento do projeto
